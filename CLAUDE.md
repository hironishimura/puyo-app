## ぷよぷよアプリ 要件定義

### プロジェクト概要

モダンで先進的な UI/UX を持つブラウザ動作のぷよぷよゲーム。  
Next.js 16 (App Router) + TypeScript + Tailwind CSS v4 + Zustand + Framer Motion で構築する。

---

### 技術スタック

| 役割 | 採用技術 |
|------|---------|
| フレームワーク | Next.js 16 (App Router) |
| 言語 | TypeScript 5 |
| スタイリング | Tailwind CSS v4 |
| 状態管理 | Zustand 5 |
| アニメーション | Framer Motion 12 / CSS Animations |
| レンダリング | React 19 (DOM ベース、Canvas 不使用) |

### ゲームルール

#### フィールド
- 縦 13 行 × 横 6 列（1 行目は隠し行、実表示は 12 行）
- ぷよは下から積み上がる

#### ぷよの種類
- 通常ぷよ：赤・青・緑・黄・紫の 5 色
- おじゃまぷよ：連鎖の相殺・攻撃に使用

#### 操作
| 操作 | キー |
|------|------|
| 左移動 | ← / A |
| 右移動 | → / D |
| 左回転 | Z / , |
| 右回転 | X / . |
| 高速落下（ソフトドロップ） | ↓ / S |
| 即時落下（ハードドロップ） | Space |

- 操作対象は「ツモ」（2 個 1 組のぷよペア）
- ツモは縦・横・逆縦の 4 方向に回転可能

#### 消去ルール
- 同色のぷよが上下左右に 4 個以上繋がると消滅
- 消滅後に浮いたぷよは落下し、再度消去判定（連鎖）

#### 連鎖・スコア
- 連鎖数に応じてスコア倍率が増加
- スコア = 消したぷよ数 × 連鎖ボーナス × 色ボーナス × グループボーナス
- おじゃまぷよは連鎖数に応じて相手フィールドに送出（対戦モード）

#### ゲームオーバー
- 出現位置（3 列目・1 行目）がぷよで埋まった時点でゲームオーバー

#### ネクスト表示
- 次のツモ（ネクスト）を 2 セット先まで表示

### 機能要件

#### 必須機能（MVP）
1. **ソロプレイモード** — 1 人でエンドレスにプレイ、スコアを記録
2. **ゲームフィールド描画** — CSS Grid / Flexbox によるセル表示
3. **リアルタイム操作** — キーボード入力でツモを操作
4. **落下・固定ロジック** — 重力による自動落下と固定タイミング制御
5. **消去・連鎖ロジック** — 4 個以上の同色検出、BFS/DFS による連鎖処理
6. **スコア表示** — 連鎖倍率込みのスコアをリアルタイム表示
7. **ネクスト表示** — 次の 2 ツモを表示
8. **ゲームオーバー画面** — スコアとリスタートボタン

#### 拡張機能（Phase 2）
- ローカルハイスコアランキング（localStorage）
- BGM / SE（Web Audio API）
- ゴーストぷよ（落下先のプレビュー）
- アクセシビリティ対応（キーリマップ、カラーブラインドモード）

---

### 非機能要件

- **パフォーマンス**：60 fps のゲームループ（`requestAnimationFrame` ベース）
- **レスポンシブ**：モバイル縦画面（375px 以上）〜デスクトップに対応
- **アクセシビリティ**：キーボードのみで全操作可能
- **ブラウザ対応**：Chrome / Safari / Firefox 最新版

### UI/UX 要件

#### ビジュアルコンセプト
- **ダークグラスモーフィズム**をベースとした近未来的デザイン
- ぷよは丸みのある SVG 風シェイプ（CSS `border-radius` + グラデーション）
- フィールド背景はブラー付き半透明パネル

#### アニメーション仕様
| イベント | アニメーション |
|---------|--------------|
| ツモ落下 | スムーズな CSS transition |
| ぷよ固定 | 小さなバウンス（scale: 1→1.1→1） |
| 消滅 | フェードアウト + パーティクルエフェクト |
| 連鎖テキスト | 中央に大きくポップイン表示（Framer Motion） |
| ハードドロップ | フラッシュ + 衝撃波エフェクト |
| ゲームオーバー | フィールド全体のシェイク + モーダル表示 |

#### レイアウト
```
┌─────────────────────────────┐
│  SCORE   │  FIELD  │  NEXT  │
│  LEVEL   │ 6 × 12  │ ネクスト│
│  CHAIN   │         │        │
└─────────────────────────────┘
```

### アーキテクチャ

#### ディレクトリ構成（予定）

```
src/
├── app/
│   ├── layout.tsx          # ルートレイアウト
│   └── page.tsx            # ゲーム画面エントリ
├── components/
│   ├── GameBoard.tsx        # フィールド描画
│   ├── PuyoCell.tsx         # 個々のぷよセル
│   ├── NextPuyo.tsx         # ネクスト表示
│   ├── ScorePanel.tsx       # スコア・レベル表示
│   ├── ChainLabel.tsx       # 連鎖数アニメーション
│   └── GameOverModal.tsx    # ゲームオーバー画面
├── store/
│   └── gameStore.ts         # Zustand ストア（ゲーム全状態）
├── game/
│   ├── constants.ts         # BOARD_WIDTH/HEIGHT、色定義など
│   ├── types.ts             # Puyo, Board, GameState 型定義
│   ├── logic.ts             # 落下・固定・消去・連鎖の純粋関数
│   ├── generator.ts         # ツモ生成（乱数シード管理）
│   └── loop.ts              # requestAnimationFrame ゲームループ
└── hooks/
    ├── useGameLoop.ts       # ゲームループを React に接続
    └── useKeyboard.ts       # キーボード入力ハンドラ
```

#### 状態管理方針
- **ゲームロジック**（`game/`）は純粋関数のみ — React/Zustand に依存しない
- **Zustand ストア**（`store/gameStore.ts`）がゲーム状態の唯一の真実
- コンポーネントはストアを購読するだけで、ロジックを持たない

#### ゲームループ
```
requestAnimationFrame
  └─ loop.ts: delta 時間計算
       └─ logic.ts: 落下判定 → 固定判定 → 消去判定 → 連鎖処理
            └─ gameStore.setState(): React 再レンダリングをトリガ
```
